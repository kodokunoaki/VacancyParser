# AGENTS.md
 
Правила для AI-агентов (Codex, Claude, Cursor и др.) и разработчиков проекта.  
Следуй этим инструкциям при любых изменениях кода.
 
---
 
## Стиль кода
 
- Форматтер: **[black](https://black.readthedocs.io/)** — не обсуждается, применяется ко всем `.py` файлам.
- Перед коммитом выполни: `black app/ tests/`
- Строки не длиннее 88 символов (дефолт black).
- Типизация обязательна: аннотируй аргументы и возвращаемые значения функций.
---
 
## Структура проекта
 
Проект модульный. Весь исполняемый код живёт в директории `app/`.
 
```
app/
├── __init__.py
├── main.py          # Точка входа — здесь только оркестрация, логика вынесена в модули
├── schemas.py       # Pydantic-схемы всех сущностей
└── core/
    ├── __init__.py
    ├── config.py    # Настройки через pydantic-settings
    └── utils.py     # Вспомогательные функции (хелперы без побочных эффектов)
```
 
**Нельзя:**
- Класть бизнес-логику напрямую в `main.py`.
- Создавать `.py`-файлы вне `app/` и `tests/` без согласования.
- Дублировать хелперы — общие функции идут в `app/core/utils.py`.
---
 
## Конфигурация (`app/core/config.py`)
 
- Все настройки описываются как `pydantic-settings` модель `Settings`.
- Значения берутся из `.env` или переменных окружения.
- Единственный экземпляр настроек: `settings = Settings()` в конце файла.
```python
# Правильный импорт настроек
from app.core.config import settings
```
 
Никаких `os.environ.get(...)` или захардкоженных значений за пределами `config.py`.
 
---
 
## Схемы данных (`app/schemas.py`)
 
- Все сущности, которыми обменивается код (вакансия, результат поиска и т.д.),  
  описываются как **Pydantic**-модели в `app/schemas.py`.
- Не используй `dict` там, где можно использовать схему.
---
 
## Импорты
 
Используй абсолютные импорты от корня пакета `app`:
 
```python
# ✅ Правильно
from app.core.config import settings
from app.core.utils import safe_text
from app.schemas import Vacancy
 
# ❌ Неправильно
from core.config import settings
from .utils import safe_text
import parser
```

## Точка входа
 
Приложение запускается командой:
 
```bash
python -m app.main
```
 
`app/main.py` должен содержать:
 
```python
if __name__ == "__main__":
    main()
```

## Тесты
 
- Тесты живут в `tests/`, файлы называются `test_*.py`.
- **Любой новый функционал** должен сопровождаться тестом.
- Запуск: `pytest tests/`
- Для юнит-тестов мокай Selenium (`unittest.mock` или `pytest-mock`) — не ходи в сеть в тестах.
```python
# Пример структуры теста
def test_safe_text_returns_default_when_element_missing():
    ...
```
